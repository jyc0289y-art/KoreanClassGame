# 맵 시스템 구현 보고서

**프로젝트**: 안녕, 서울 (Korean Class RPG Game)
**작업일**: 2026-02-27
**작업범위**: 3계층 맵 시스템 전체 구현 + 장소맵 상세화

---

## 1. 작업 개요

Phaser 3.90.0 + Vite 6.x 기반 한국어 학습 RPG 게임에 실제 지리 기반 3계층 맵 시스템을 구축하였다. 기존에는 FukuokaScene과 IncheonScene 2개의 단일 월드맵만 존재했으나, 이번 작업으로 **28개 씬**을 가진 완전한 맵 네비게이션 체계를 구현하였다.

모든 장소맵은 웹 조사를 통해 실제 시설의 레이아웃, 내부 구조, 주요 시설 배치를 참고하여 제작하였다.

---

## 2. 시스템 아키텍처

### 2-1. 3계층 맵 구조

```
┌─────────────────────────────────────────────────────┐
│  Layer 1: 국제맵 (InternationalMapScene)            │
│  ─ UI 선택 화면: 동아시아 지도에서 공항 선택          │
│  ─ 후쿠오카공항 ↔ 인천공항 비행 애니메이션            │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────┴──────────────────────────────┐
│  Layer 2: 광역맵 (MetroMapScene)                    │
│  ─ UI 선택 화면: 지하철 노선도에서 역 선택           │
│  ─ FukuokaMetroScene / SeoulMetroScene              │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────┴──────────────────────────────┐
│  Layer 3a: 지역맵 (BaseWorldScene 상속)             │
│  ─ 워커블 2D 월드 (조이스틱/키보드 이동)             │
│  ─ NPC, 건물, 지하철역 입구, 장소 입구 포함          │
├─────────────────────────────────────────────────────┤
│  Layer 3b: 장소맵 (BasePlaceScene 상속)             │
│  ─ 실내/특정 장소의 워커블 맵                        │
│  ─ NPC 대화, 미션 시스템 연동                        │
└─────────────────────────────────────────────────────┘
```

### 2-2. 이동 흐름 예시

```
후쿠오카 야쿠인 → 야쿠인역 🚇 → 후쿠오카 광역맵 → 후쿠오카공항역
→ 후쿠오카공항 → 국제선 탑승구 ✈ → 국제맵 → 인천공항 선택
→ 인천공항 도착 → 공항철도역 🚇 → 서울 광역맵 → 명동역
→ 명동 지역맵 → 올리브영 건물 진입 → 올리브영 장소맵
```

## 3. 신규 베이스 클래스

### 3-1. BaseMapUIScene (국제맵/광역맵 공통)
- 워커블이 아닌 UI 선택 화면
- Phaser Graphics로 지도/노선도 프로시저럴 렌더링
- 노드(역/공항) 원형 버튼 배치, 터치/클릭 선택
- 레벨 게이팅: 잠긴 역은 자물쇠 표시
- 현재 위치 표시 (깜빡이는 핑크 점)

### 3-2. BasePlaceScene (장소맵 공통)
- BaseWorldScene 상속
- 실내용 타일맵 (벽/바닥 자동 생성)
- 출구 문 → 원래 지역맵으로 자동 복귀
- 미니맵 비활성화 (실내 불필요)
- 카메라 줌 1.0 고정

## 6. 기술 구현 사항

### 6-1. 프로시저럴 렌더링
모든 맵은 외부 스프라이트 에셋 없이 **Phaser Graphics API**와 **이모지 텍스트**만으로 렌더링한다.

```javascript
// 예시: 불판 테이블 렌더링
g.fillStyle(0x8B6914, 0.5);           // 나무 테이블
g.fillRoundedRect(x-70, y-35, 140, 90, 8);
g.fillStyle(0x333333, 0.7);           // 불판 (검은 철판)
g.fillCircle(x, y, 25);
g.fillStyle(0xFF4500, 0.25);          // 불꽃 표현
g.fillCircle(x, y, 20);
```

### 6-2. createPlace() 설정 패턴
BasePlaceScene의 표준 생성 패턴:

```javascript
this.createPlace({
  worldWidth: 800, worldHeight: 700,
  startX: 400, startY: 640,
  tiles: 'floor_tile',
  returnScene: 'SeoulMyeongdongScene',
  title_ko: '게스트하우스', title_ja: 'ゲストハウス',
  subtitle: '명동 숙소',
  npcs: [ /* NPC 배열 */ ],
  buildings: []
});
```

### 6-3. 네비게이션 시스템
BaseWorldScene에 추가된 이동 메서드:

| 메서드 | 용도 |
|--------|------|
| `createSubwayEntrance(x, y, metro, station)` | 지하철역 입구 → 광역맵 |
| `createEnterableBuilding(x, y, scene, config)` | 건물 진입 → 장소맵 |
| `createDepartureGate(x, y, config)` | 국제선 탑승구 → 국제맵 |

### 6-4. GameState 확장 필드

```javascript
{
  currentRegion: 'fukuoka',       // 현재 지역
  currentMap: 'fukuoka_yakuin',   // 현재 맵

  lastAirport: null,              // 마지막 이용 공항
  visitedMaps: [],                // 방문 맵 기록
  unlockedStations: []            // 해금된 역 목록
}
```
